搭建过程

当时自己在搭建博客的时候主要是跟着【这篇文章】一步步搭建的。但文章中有个地方中我觉得没有讲清楚，就是运行完hexo init blog初始化之后，所有的hexo命令基本都是在初始化生成的blog文件夹中执行的。当初自己就是在blog文件夹外面执行hexo g、hexo s等命令，都显示不存在该命令，弄了很久才发现问题。

此外，在GitHub用来托管博客的仓库名必须为【用户名.github.io】，这个用户名使用你的GitHub帐号名称代替，这是固定写法，不能用其它的名字。

博客的主题用的是NexT，基本的设置以及个性化的设置当时参考了上面提到的那篇文章、【这篇文章】以及【这篇文章】。当然关于个性化的设置，网上还能搜到很多不错的文章。

一些问题及解决方法

hexo d 部署后总需要重新改域名

在/blog/source目录下创建一个CNAME文件，然后保存成 All files 格式（即没有后缀名），CNAME文件里写自己新的域名。其实在我提到的第一篇文章中也有提到这个，这里我主要是想说明一下怎么保存成 All files 格式：先新建一个文本文件，在里面写自己的域名，另存为该文件，保存类型选所有文件，文件名填CNAME.，注意要加个点。

本地预览和发布的效果不一样

发布前要先清除一下缓冲（即运行hexo clean），然后再hexo g、hexo d。此外可能还要等一会才会发生变化。
